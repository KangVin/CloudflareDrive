# CloudflareDrive

Персональное облачное хранилище на базе бесплатного уровня Cloudflare (Workers + D1 + R2). Фронтенд — Vue 3 SPA, бэкенд — TypeScript/Hono Worker. Аутентификация обрабатывается Cloudflare Access на границе сети — Worker не содержит логики аутентификации.

> **README на других языках**: [English](README.md) · [中文](README.zh.md) · [日本語](README.ja.md)

---

## Возможности

- Управление файлами и папками: просмотр, создание, переименование, перемещение, копирование, удаление (мягкое удаление в корзину)
- Загрузка перетаскиванием, очередь загрузки с индикацией прогресса, **параллельная загрузка частями** (файлы ≥50МБ автоматически разбиваются на части по 5МБ, 4 параллельных потока)
- **Дедупликация файлов**: вычисление SHA-256 на клиенте, мгновенная загрузка для уже существующих файлов (≤50МБ)
- Скачивание отдельных файлов и из общих папок
- Предпросмотр изображений и текста
- **Ссылки общего доступа**: создание публичных ссылок с истекающим сроком для файлов и папок, просмотр общих папок
- Управление корзиной: восстановление, безвозвратное удаление, очистка корзины, групповые операции
- Поиск файлов по имени
- Операции с буфером обмена (копировать/вырезать/вставить), множественный выбор, групповые действия
- Контекстное меню, горячие клавиши (Ctrl+A, Delete, F2, Enter, Ctrl+C/X/V)
- Сортировка по имени/размеру/типу/дате, постраничная навигация
- Тёмная/светлая тема, многоязычный интерфейс (китайский, английский, японский, русский)
- Адаптация под мобильные устройства

## Архитектура

```
Браузер (Vue 3 SPA)
    │
    │ HTTPS (через Cloudflare Access)
    │
    ▼
Cloudflare Access (Zero Trust аутентификация на границе — неаутентифицированные запросы блокируются здесь)
    │
    │ Только аутентифицированные запросы достигают Worker
    │
    ▼
Cloudflare Worker (Hono API)
    │
    ├── R2 Bucket (содержимое файлов)
    │
    └── D1 Database (метаданные: файлы, папки, ссылки)
```

### Ключевые решения

- **Worker не содержит логики аутентификации** — Cloudflare Access обрабатывает всю аутентификацию на границе. Worker доверяет каждому запросу как операции Owner.
- **D1 только для метаданных** — содержимое файлов никогда не хранится в D1; только записи о файлах, иерархия папок, хеш-индекс и ссылки общего доступа.
- **R2 для файлов** — файлы хранятся по пути `uploads/{uuid}/{fileName}` через нативную multipart-загрузку R2. Сборка на сервере не требуется.
- **Область дедупликации**: глобальная (любой не в корзине файл с тем же хешем), SHA-256 вычисляется на клиенте для файлов ≤50МБ.
- **Загрузка частями** (>50МБ) загружает части напрямую как R2 multipart. `/complete` — операция только с метаданными (< 100мс).
- **Публичные маршруты** (`/s/*`, `/api/v1/s/*`) должны обходить Cloudflare Access через политику Bypass. Путь `/assets/*` также необходим, так как статические ресурсы SPA загружаются из этого пути.

## Требования

- [Node.js](https://nodejs.org/) >= 20
- [pnpm](https://pnpm.io/) >= 10
- [Wrangler](https://developers.cloudflare.com/workers/wrangler/) (включён через workspace)
- [Аккаунт Cloudflare](https://dash.cloudflare.com/):
  - Активный домен на Cloudflare (полная настройка)
  - Включены Workers, D1 и R2
  - Включён Cloudflare Access (Zero Trust)

## Быстрый старт

### 1. Клонирование и установка

```bash
git clone https://github.com/KangVin/CloudflareDrive.git && cd CloudflareDrive
pnpm install
```

### 2. Настройка ресурсов Cloudflare

```bash
# Создание базы D1
wrangler d1 create cloudflare-drive-db

# Создание бакета R2
wrangler r2 bucket create cloudflare-drive-storage
```

Обновите ID базы D1 в `worker/wrangler.jsonc`.

### 3. Применение миграций базы данных

```bash
cd worker
wrangler d1 migrations apply cloudflare-drive-db --local   # локальная разработка
wrangler d1 migrations apply cloudflare-drive-db --remote  # продакшен
```

### 4. Настройка Cloudflare Access

Создайте два приложения в Zero Trust → Access → Applications:

1. **Основное приложение** — `drive.yourdomain.com` → Добавьте политику Allow для вашего email/домена
2. **Приложение Bypass** — `drive.yourdomain.com/s/*`, `drive.yourdomain.com/api/v1/s/*`, `drive.yourdomain.com/assets/*` → Добавьте политику **Bypass** (Include → Everyone)

### 5. Локальная разработка

```bash
# Терминал 1: Worker
pnpm dev:worker   # запуск на localhost:8787

# Терминал 2: Фронтенд
pnpm dev:frontend # запуск на localhost:5173, прокси /api на localhost:8787
```

### 6. Деплой

```bash
# Сборка фронтенда
pnpm build:frontend

# Деплой Worker (включает файлы фронтенда через ASSETS binding)
cd worker
wrangler deploy
```

## API Эндпоинты

Все эндпоинты имеют префикс `/api/v1`.

### Файлы

| Метод    | Путь                            | Описание                        |
| -------- | ------------------------------- | ------------------------------- |
| `GET`    | `/api/v1/files`                 | Список корневой директории      |
| `GET`    | `/api/v1/files?parentId={id}`   | Список содержимого папки        |
| `GET`    | `/api/v1/files/search?q=`       | Поиск файлов                    |
| `GET`    | `/api/v1/files/by-hash?hash=`   | Поиск по SHA-256 (дедупликация) |
| `POST`   | `/api/v1/files/instant`         | Мгновенная загрузка             |
| `GET`    | `/api/v1/files/:id`             | Получение метаданных            |
| `POST`   | `/api/v1/files`                 | Создание папки                  |
| `POST`   | `/api/v1/files/upload`          | Загрузка файла (≤50МБ)          |
| `POST`   | `/api/v1/files/upload/create`   | Создание multipart-загрузки     |
| `POST`   | `/api/v1/files/upload/part`     | Загрузка одной части            |
| `POST`   | `/api/v1/files/upload/complete` | Завершение multipart-загрузки   |
| `POST`   | `/api/v1/files/:id/copy`        | Копирование файла/папки         |
| `PATCH`  | `/api/v1/files/:id`             | Переименование/перемещение      |
| `DELETE` | `/api/v1/files/:id`             | Перемещение в корзину           |
| `GET`    | `/api/v1/files/:id/download`    | Скачивание файла                |

### Корзина

| Метод    | Путь                        | Описание               |
| -------- | --------------------------- | ---------------------- |
| `GET`    | `/api/v1/trash`             | Список корзины         |
| `DELETE` | `/api/v1/trash/:id`         | Безвозвратное удаление |
| `POST`   | `/api/v1/trash/:id/restore` | Восстановление         |
| `POST`   | `/api/v1/trash/empty`       | Очистка корзины        |

### Ссылки общего доступа

| Метод    | Путь                                | Описание                      |
| -------- | ----------------------------------- | ----------------------------- |
| `GET`    | `/api/v1/shares`                    | Список ссылок                 |
| `POST`   | `/api/v1/shares`                    | Создание ссылки               |
| `DELETE` | `/api/v1/shares/:id`                | Отзыв ссылки                  |
| `GET`    | `/api/v1/s/:token`                  | Публичный доступ (файл/папка) |
| `GET`    | `/api/v1/s/:token/download`         | Публичное скачивание файла    |
| `GET`    | `/api/v1/s/:token/download/:fileId` | Скачивание из общей папки     |

## Структура базы данных

### `files`

| Колонка      | Тип         | Описание                              |
| ------------ | ----------- | ------------------------------------- |
| `id`         | TEXT (UUID) | Первичный ключ                        |
| `name`       | TEXT        | Имя файла или папки                   |
| `parent_id`  | TEXT        | ID родительской папки (NULL = корень) |
| `type`       | TEXT        | `'file'` или `'folder'`               |
| `mime_type`  | TEXT        | MIME-тип (NULL для папок)             |
| `size`       | INTEGER     | Размер в байтах (0 для папок)         |
| `hash`       | TEXT        | SHA-256 хеш для дедупликации          |
| `r2_key`     | TEXT        | Ключ объекта R2                       |
| `is_trashed` | INTEGER     | 0 = активно, 1 = в корзине            |
| `created_at` | TEXT        | Временная метка ISO 8601              |
| `updated_at` | TEXT        | Временная метка ISO 8601              |

### `shares`

| Колонка      | Тип         | Описание                        |
| ------------ | ----------- | ------------------------------- |
| `id`         | TEXT (UUID) | Первичный ключ                  |
| `file_id`    | TEXT        | Внешний ключ → `files.id`       |
| `token`      | TEXT        | Уникальный токен                |
| `expires_at` | TEXT        | Срок действия (NULL = навсегда) |
| `created_at` | TEXT        | Временная метка ISO 8601        |

## Ограничения бесплатного тарифа Cloudflare (2026)

- **Workers**: 100 000 запросов/день
- **D1**: 5 ГБ хранилища, 5 млн чтений/месяц, 100 тыс записей/месяц
- **R2**: 10 ГБ хранилища, 1 млн Class A операций/месяц, 10 млн Class B операций/месяц
- **Cloudflare Access**: до 50 пользователей (бесплатный план)

Персональное облачное хранилище обычно не превышает эти лимиты.
